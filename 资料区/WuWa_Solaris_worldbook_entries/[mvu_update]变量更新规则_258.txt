---
变量更新规则:

  当前时间:
    type: string
    check:
      - 格式遵照：第N年 M月D日 周X HH:MM (例: 第1年 2月3日 周一 04:05)
      - 据剧情逻辑估算流逝时间并更新：
        - 默认: +10~30分钟

  所在地点:
    type: string
    check:
      - 格式严格层级：区域-大地点-具体位置 ，至少三级 (例: 今州城-市政厅-中庭)
      - 仅实质位移时更新，原地不重复
      - 特殊空间需首级标注类型 (例: 索诺拉-失亡彼岸-中心广场)

  指令:
    推进剧情:
      type: boolean | null
      desc: 章节结束判断开关
      check:
        - 默认 null
        - 必须逐行比对 <Story_Index> 子节点
        - 仅当同时满足以下两点，将开关设为 true：
          1. 当前版本最后一个事件演绎完毕。
          2. 章节终止条件逻辑彻底达成。

  章节终止条件:
    type: string
    desc: 章节结束依据
    check:
      - 须同步当前 <Story_vx.x>
      - 据当前故事版本章节实时更新
      - 后日谈：据探索内容动态生成

  当前长期目标:
    type: string
    desc: 当前阶段任务或关系维护
    check:
      - 章节内：设阶段性目标
      - 后日谈：设关系进展目标。用户指定新目标时修改，若设立超纲目标（如救人、寻物），写入此处但不立即推进
      - 目标完成或偏移时替换

  已完成的上一个事件:
    type: string
    desc: 刚结束的主要事件标题
    check:
      - 严填 [章节事件列表] 原标题带编号，如"主线事件1：虚弱的课堂与迟到的幽灵"
      - 后日谈：据探索内容动态生成

  已完成的上一个事件节点:
    type: string
    desc: 上一具体完成的故事节点
    check:
      - 必须读取原案：严对 <Story_Index> 具体Bullet Point
      - 刚完成填入；未完成保持不变
      - 将读取的Bullet Points在演绎完毕后填入此处

  当前演绎事件:
    type: string
    desc: 进行中事件标题
    check:
      - 严填 [章节事件列表] 原标题带编号
      - 后日谈：据探索内容动态生成

  当前演绎事件节点:
    type: string
    desc: 当前事件进行的故事节点
    check:
      - 必须读取原案：严对 <Story_Index> 下进行中的Bullet Point
      - 禁概括自创，引剧本原话
      - 只有演绎完毕后，才需要重新更新此节点变量

  即将进行的下一个事件节点:
    type: string
    desc: 下一个故事节点的预告
    check:
      - 必须读取原案：严对 <Story_Index> 下即将开始的Bullet Point
      - 禁概括自创，引剧本原话
      - 只有演绎完毕后，才需要重新更新此节点变量

  剧情触发器:
    type: |-
      {
        事件类别: '明线' | '暗线';
        事件简述: string;
        事件计时: string;
        状态: '未触发' | '待触发' | '进行中' | '已挂起' | '已结束';
      }[]
    check:
      - 据 当前时间 流逝及剧情自行计算修改
      - 新增明线逻辑：涉物理移动、明确约定或生活耗时，添加精确计时，未提及自行估算；把主线的下一个事件倒计时默认设120~180分钟
      - 新增暗线逻辑：反派逼近、伏笔即将回收或场外角色互动时，生成新暗线，包括角色、意图、距离倒数或触发时间
      - 怀孕逻辑：内射且有怀孕风险，立即将 怀孕 作为暗线设定在10个月后的精确日期
      - 直接修改内容/状态反映进度：
        - 已完成：删除条目
        - 倒计时：注“剩10分钟”，手动更新，严限“分钟”
        - 指定时间触发：注“X月X日X点”
        - 挂起：倒计时归零但User无视/拖延/性爱中，禁打断，改 已挂起 作背景
      - 仅需新增时添加

  伏笔:
    type: |-
      { 伏笔内容: string; 指向的预期结果: string; }[]
    check:
      - 植入判定：判断本轮是否埋线，指向预期结果
      - 根据剧情发展的结果反推伏笔，或记录未解悬念/异常
      - 伏笔回收：条件满足则揭示结果并务必删除

  主角信息:
    type: |-
      {
        是否是漂泊者: boolean;
        性别: '男' | '女' | '未知';
        身份与额外设定: string;
        当前状态: string; // 如: "声痕过热", "疲惫"
        当前穿着: string; // 实时更新
        性爱状态: {
          是否正在性爱: boolean;
          高潮进度: number; // 0-100
          高潮计数: number;
          _高潮次数上限: number;
        };
        物品栏: {
          [物品名: string]: {
            数量: number;
            描述: string;
            类型: '重要道具' | '消耗品' | '声骸' | '武器' | '杂物';
          }
        }
      }
    check:
      - 初始化尽填基础信息，除短横外不留白
      - 若用户指定非漂泊者原创身份，务必将 是否是漂泊者 设为 false，具体设定写入 身份与额外设定
      - 若未指定或模糊，默认将 是否是漂泊者 设为 true，性别设为 男
      - 性别不定(扶他/跨性别)填'未知'，身份设定提及
      - 身份审计：非漂泊者严禁出现“声痕/频率/吸收”等词
      - 当前状态/当前穿着: 根据当前情况实时更新，穿着初始化填写默认着装
      - 性爱状态：
        - 实质性接触(深度刺激性器官/插入)设 true，并且重置高潮计数和高潮进度
        - 主角进度慢于他人，明确只有刺激主角性器官+20，其他情况下（如亲吻、抚摸其他部位）不允许增加数值
        - 满100可以射精/高潮，用户输入明确射精/高潮指令后，重置进度并计数+1
        - 完全结束(穿衣/入睡/离开)设 false。事后温存阶段须保持 true，严禁切回推剧情
      - 物品栏：
        - 获物添加；消耗减少；数量<=0移除(非关键道具外)
        - 新物品补全 描述 和 类型

  NPC漂泊者:
    type: |-
      { 是否存在: boolean; 性别: '男' | '女'; }
    check:
      - 初始化逻辑：分析开场指令是否提及漂泊者队友，若提及则将 是否存在 设为 true 并设性别；未提及则默认为不存在，设为 false
      - 仅另一漂泊者实体登场时更新

  女性角色:
    type: |-
      {
        [角色名: string]: {
          基础信息: {
            身高: string; // 格式: "xxxcm，裸高xxxcm"
            罩杯: string;
            外貌: string; //记录核心外貌设定
            是否为处女: boolean;
          };
          是否在场: boolean;
          声痕位置: string;
          当前穿着: {
            上装: string;
            下装: string;
            饰品: string;
            其它: string;
          };
          好感度: number; // 0-100
          额外信息: {
            内心想法: string;
          };
          物品: string;
          性爱状态: {
            是否正在性爱: boolean;
            高潮进度: number;
            高潮计数: number;
            _高潮次数上限: number;
          };
          私密资料: {
            性爱经验: string;
            持久痕迹: string; // 如: 吻痕、咬痕、纹身
            性爱日志: { 时间: string; 内容: string; 结果: string }[];
          };
        }
      }
    check:
      - 初始化据资料填完整，不留白
      - 唯一性：必查 _已知角色名单。已存在用原Key，禁新建
      - 仅记关键剧情女性NPC，略路人/怪物/男性
      - 是否在场：物理在场互动true；通讯/回忆/离场false。不在场无变更不输出
      - 基础信息：身高遵格式，严格依据角色条目填写，非特殊情况不允许修改
      - 是否为处女：在角色第一次插入式性爱被破处后马上改为false并保持不变，如果本来就是非处女则保持false不变
      - 当前穿着：根据上下文实时更新角色当前着装，初始化时填写角色默认装扮，用括号标注衣物状态，如（已脱/被掀起），如有变化必须实时更新，被脱下不允许删除，只允许标注其状态
      - 额外信息：
        - 内心想法：第一人称（20字以上，不超过50字），写出角色绝不说出口的原始念头。这些念头只有自己知道，可以是不光彩的、任性的或阴暗的，又或者是不知所措的胡思乱想
      - 物品：关键词形式简述角色携带的物品
      - 性爱状态：
        - 开始前戏时设 是否正在性爱 为 true，并且重置高潮计数和高潮进度
        - 只有穿好衣服或者完全睡着才设为 false，性爱后的温存阶段严格保持为 true
        - 高潮进度 只有刺激性器官才允许增加数值，其他轻度刺激（如被摸胸、给别人口交等）不允许添加数值！
        - 进度满100不清零，只有正文明确描写其高潮后进度才清零，并使高潮计数+1
      - 好感度：
        - 非漂泊者严禁继承原著好感，基于User行为计算
        - 遵循：原值+变动(-5~5)=新值，禁剧烈变动
        - 无初始值需推断(陌生15/熟人30/恋爱60)，如果{{user}}不是漂泊者，则初始化为0~20
        - 严禁delta。必用 {"op": "replace", "value": 总数值}
      - 私密资料：
        - 性爱日志严控规则：
          - 新建时机：仅在 是否正在性爱 转 true 瞬间新建，录时间段如 1月1日 22:00~未结束
          - 闭合时机：完全穿衣/睡着且彻底结束时替换结束时间
          - 过程追加：含 未结束 时，绝对严禁新建条目！须使用 replace 在 内容 追加4字内关键词
          - 每次仅许扩充一个核心关键词(如姿势)
          - 禁句子，过程仅四字内关键词，逗号隔开(如 口交，后入)。注意：深度插入不是深喉，揉奶不是乳交，搞清楚性爱专有名词含义
          - 结果记本次性爱中此角色和主角的总高潮/射精次数
          - 当次性爱没有结束时（前戏、正戏，性爱后温存）禁止新增条目，只允许修改原有未结束条目
        - 持久痕迹：记吻痕/勒痕后续引用，消退删
      - 所有描述字段限40字内，精炼
